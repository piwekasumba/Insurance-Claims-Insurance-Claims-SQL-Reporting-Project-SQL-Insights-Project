##Basic ETL and Data Transformation Using SQL

This stage demonstrates a basic SQL ETL workflow using the insurance claims dataset to prepare raw data for reporting and analysis.

The workflow includes:

- Created a raw data table (`claims_raw`) to represent incoming source data
- Cleaned data by removing missing and invalid records
- Standardized text values for consistent reporting (for example, `policy_type`)
- Created derived fields including:
- Claim year for time-based reporting
- Claim risk level for business classification
- Loaded transformed data into a cleaned dataset (`claims_clean`)
- Built a reporting table (`claims_summary`) using aggregated business metrics

This workflow follows a structured Extract, Transform, Load (ETL) process that prepares data for reliable reporting, trend analysis, and business decision-making using SQL.
