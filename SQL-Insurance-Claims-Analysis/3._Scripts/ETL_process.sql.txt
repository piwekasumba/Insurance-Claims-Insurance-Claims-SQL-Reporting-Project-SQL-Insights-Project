# ETL and Data Preparation

This project includes a simplified SQL ETL (Extract, Transform, Load) workflow that prepares insurance claims data for reporting and business analysis using PostgreSQL.

## ETL Process

### Extract

- Loaded raw insurance claims data into PostgreSQL tables.
- Organised customer and claims data within a relational database structure.

### Transform

- Removed invalid and incomplete records.
- Standardised data values for consistent reporting.
- Created derived reporting fields, including:
  - Claim Year
  - Claim Month
  - Claim Severity
- Prepared clean, reporting-ready datasets using SQL transformations.

### Load

- Loaded the transformed data into reporting tables.
- Created customer and policy summary tables to support business reporting and KPI analysis.

## Reporting Value

The ETL process improves data quality and prepares structured datasets for:

- Claims reporting
- Customer claims analysis
- Policy performance reporting
- Monthly trend analysis
- KPI reporting
- Operational decision support

This demonstrates how SQL can be used to transform raw business data into reliable, reporting-ready information that supports business decision-making.
