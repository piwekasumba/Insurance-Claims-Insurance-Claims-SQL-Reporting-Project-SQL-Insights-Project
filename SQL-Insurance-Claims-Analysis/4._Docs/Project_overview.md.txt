 SQL Insurance Claims Analysis Project

-- Project Overview

This project analyzes insurance claims data using PostgreSQL to explore customer behaviour, claim activity, and business risk. The aim was to practice the complete SQL analysis process, from preparing and cleaning the data to producing insights that could support reporting and business decisions.

The project simulates work commonly performed by a Junior Data Analyst, using SQL to transform raw data into clear and meaningful information.

-- Objectives

- Build a relational database for insurance claims data
- Clean and prepare data for analysis
- Write SQL queries to answer business questions
- Analyse customer and claim trends
- Produce reporting-ready outputs using SQL

-- Dataset

The project uses two related tables:

- Customers – customer demographic and lifestyle information
- Claims – insurance claim records linked to each customer

The tables are connected using customer_id, allowing customer and claim data to be analysed together.

-- SQL Work Completed

- Created relational database tables
- Cleaned and standardised the data
- Established relationships using primary and foreign keys
- Joined multiple tables for analysis
- Used aggregate functions and GROUP BY to summarise data
- Created calculated fields for reporting
- Built summary queries to support business reporting

-- Business Questions Explored

• Which customer groups submit the most claims?
• Which claim severity levels account for the highest claim amounts?
• How do claims change over time by year and month?
• Which customers and policies generate the highest claim values?
• What trends could help support business reporting and risk monitoring?
