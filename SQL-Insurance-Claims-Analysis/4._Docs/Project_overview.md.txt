SQL Insurance Claims Analysis Project
Project Overview

This project demonstrates how I used PostgreSQL to analyse a simulated insurance claims dataset from a business reporting perspective.

The project focuses on organising relational data, writing SQL queries to answer business questions, calculating reporting metrics, and producing reporting-ready outputs that support operational monitoring and decision-making.

It reflects the type of analytical thinking expected from an entry-level Reporting Analyst working with structured business data.

Business Scenario

Insurance companies process large volumes of claims data every day. Without accurate reporting, it becomes difficult to monitor claims activity, identify trends, measure operational performance, and support business decisions.

This project demonstrates how SQL can transform raw claims data into meaningful information for reporting and business analysis.

Project Objectives:

Build a relational database using PostgreSQL
Prepare structured data for analysis
Write SQL queries to answer business questions
Analyse claims activity and reporting metrics
Produce reporting-ready datasets
Support data-driven business decisions
